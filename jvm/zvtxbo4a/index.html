i++和++i

问题：int i = 0; i = i++; 最终i的值是多少？

答案是显而易见的，i的值是0。因为i++是先赋值再自增，所以i = i++的结果是i先赋值给i，然后i再自增，所以i的值还是0

那么问题来了，为什么"i = i++的结果是i先赋值给i，然后i再自增"呢？

这道题其实用字节码的知识来解释会更加完美。问题代码对应的字节码指令如下：

iconst_0
istore_1
iload_1
iinc 1 by 1
istore_1
return

从字节码指令可以看出，i = i++的过程可以分为以下几步：

  1. 将常量0压入操作数栈
  2. 将操作数栈顶的值赋给局部变量表中的变量i
  3. 将局部变量表中的变量i的值压入操作数栈
  4. 将局部变量表1号位置的的数据加1
  5. 将操作数栈顶的值赋给局部变量表中的变量i

可以看到，i++是给局部变量表中的数字自增了，但是并没有改变操作数栈中的值

而对于++i，将上面的字节码指令的第3步和第4步调换了一下顺序，分析过程同理

类的初始化

问题1

问题：下面代码的输出结果是什么？

public class Test1 {
    public static void main(String[] args) {
        System.out.println("A");
        new Test1();
        new Test1();
    }

    public Test1(){
        System.out.println("B");
    }

    {
        System.out.println("C");
    }

    static {
        System.out.println("D");
    }
}

答案：DACBCB

这里先说明一个前置知识：静态代码块和普通代码块

对于带有static修饰的静态代码快，会在类被实例化之前就执行代码块中的内容，并且只会执行一次

而对于没有任何修饰的普通代码块，会在每次实例化对象的时候执行一次

分析过程：

  1. 首先会执行静态代码块中的内容(对应字节码中的<clint>)，打印D
  2. 然后就会执行main方法中的第一句输出语句，打印A
  3. 然后执行new Test1()，会先执行普通代码块中的内容，打印C，然后执行构造方法中的内容，打印B。最终执行了两遍new Test1()

问题2

问题：下面代码的输出结果是什么？

public class Demo01 {
    public static void main(String[] args) {
        new B02();
        System.out.println(B02.a);
    }
}

class A02 {
    static int a = 0;
    static {
        a = 1;
    }
}

class B02 extends A02{
    static {
        a = 2;
    }
}

答案：2

分析过程：

  1. 首先执行main方法中的new B02()，而B02类是继承自A02类的，所以会先执行A02类的静态代码块，将a的值设置为1
  2. 初始化父类完毕之后，再执行B02类的静态代码块，将a的值设置为2

所以，最终的结果就是2了

那如果这里变化一下题目，把new B02()给注释掉，那么结果又是多少呢？

结果不再是2了，结果是1。下面来分析一下：

  1. 进入main方法之后，直接执行System.out.println(B02.a);。这个时候，虚拟机尝试访问B02中的静态变量a
  2. Java虚拟机会首先检查 B02 的直接父类 A02，发现静态字段 a 是在 A02 中定义的
  3. 由于 B02 没有被直接引用，其 static 代码块不会被执行，所以a的值保持为1，并打印输出

问题3

问题：下面代码的输出结果是什么？

public class Test2 {
    public static void main(String[] args) {
        Test2_A[] arr = new Test2_A[10];

    }
}

class Test2_A {
    static {
        System.out.println("Test2 A的静态代码块运行");
    }
}

结果是什么都不会输出。因为这里只是创建了一个数组，并没有实例化对象，静态代码块不执行
