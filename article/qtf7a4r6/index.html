Github自动部署脚本

510字约2分钟

教程

2024-09-05

0.前言

本文将会介绍如何自动化地将VuePress/VitePress项目部署到Github Pages上

1.创建脚本文件

在项目的根目录下(和package.json文件同级)创建一个文件，文件名叫做deploy.sh，然后把下面的代码复制进去：

#!/usr/bin/env sh
 
# 忽略错误
set -e  #有错误抛出错误
 
# 构建
pnpm run docs:build  #然后执行打包命令
 
# 进入待发布的目录
cd docs/.vuepress/dist  #进到dist目录
 
git init  #执行这些git命令
git add -A
git commit -m 'deploy'
 
git push -f https://github.com/example/example.github.io.git master:gh-pages  #提交到这个分支
 
cd -
 
rm -rf docs/.vuepress/dist  #删除dist文件夹

提示

  1. 上面的示例代码用的是pnpm包管理器，如果你用的是其它包管理器，比如"yarn"，那么需要将pnpm run docs:build改成yarn run docs:build

  2. 如果你的项目是VitePress，那么需要将上面所有的vuepress替换成vitepress

  3. 把仓库的地址改成自己的仓库

2.修改package.json文件

在package.json文件中的script字段新添加一条脚本，修改后的文件如下：

"scripts": {
    "deploy":"bash deploy.sh",
    "docs:dev": "vuepress dev docs",
    "docs:dev-clean": "vuepress dev docs --clean-cache --clean-temp",
    "docs:build": "vuepress build docs --clean-cache --clean-temp",
    "docs:preview": "http-server docs/.vuepress/dist",
    "vp-update": "vp-update"
},

其中，"deploy":"bash deploy.sh",是新加进去的命令

如果是vitepress项目，其它命令会有所差别但是新加的这条命令都是一样的

3.执行自动部署脚本

完成了脚本的编写和文件配置之后，接下来可以开始执行脚本了

首先确保你已经在Github上有一个叫做example.github.io的仓库

然后打开Git Bash，通过cd命令进入你的项目的根目录下，然后输入如下命令：

pnpm run deploy

注意

如果你用的包管理器不是pnpm请自行替换

接下来，静待几秒钟，你的项目就会自动部署到对应的Github仓库中去了

优雅，无需多言😎
