Spring杂记 ​

前言 ​

这部分杂记类似于随笔，并不是系统学习时候的笔记，而是从平时编程碰到的一些问题中总结出来的知识点

虽然已经系统地学习过了SpringBoot，但是掌握的还不够扎实，通过记笔记📒的方式复习一下

Spring的配置 ​

组件的扫描、自动装配、组件管理是Spring框架非常厉害的地方😎

默认情况下，Spring的启动类只会扫描启动类所在包下的组件

但是，很多时候，尤其是在微服务的项目架构下，经常要用到跨模块之间的组件调用

来举个🌰:

现在在一个电商微服务架构的项目下面有三个模块:

  • Common模块，用于定义一些通用的工具类以及各种配置类
  • Cart模块，处理一些和电商购物车🛒相关的业务逻辑
  • GateWay模块，也就是网关模块，主要处理用户登录，拦截部分请求🫸

现在我们在Common模块中有一个用于注册全局拦截器的配置类:MvcConfig，代码如下:

java
@Configuration
public class MvcConfig implements WebMvcConfigurer {
    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(new UserInfoInterceptor());
    }
}

这个配置类注册了一个全局拦截器实例对象new UserInfoInterceptor()

但是要想让这个全局拦截器实现跨模块拦截，显然一个@Configuration是不够的，它只作用于自己的模块

因此，为了让其它类也能应用这个拦截器，有如下方案:

1.在spring.factories中配置一下配置类的包路径

spring.factories 文件通常位于项目的 META-INF 目录下（例如 src/main/resources/META-INF/spring.factories）

它定义了一个或多个 Spring Boot 自动配置的实现类。这些实现类会在 Spring Boot 应用程序启动时被自动加载并应用。

并且作用域是全局的。例如，可以这样书写配置:

org.springframework.boot.autoconfigure.EnableAutoConfiguration=\
  com.hmall.common.config.MyBatisConfig,\
  com.hmall.common.config.MvcConfig

写清楚是在哪个包下面的哪个配置类，这样就可以实现将拦截器应用于全局，这种配置方式很高效

2.用@Import注解导入

如果要让Cart这个模块也应用这个拦截器，就需要在Cart模块的启动类上进行配置导入:

java
@Configuration
@Import(com.example.common.config.CommonConfig.class)
@SpringBootApplication
public class CartServiceApplication {
    // 应用程序入口点
    public static void main(String[] args) {
        SpringApplication.run(CartServiceApplication.class, args);
    }
}

前提是这个配置类已经用@Configuration标注了才能这样导入

这种方式比较低效，因为一旦模块多了，每个模块都要导入一遍，效率太低了😭

3.@ComponentScan注解

这种方式需要在对应的配置类上面加上注解:

java
@ComponentScan({"com.hmall", "com.otherpackage",……})

后面要把所有要应用的包全都指定，非常麻烦，效率也很低😣

⚠️完成上面的步骤之后，还要特别注意网关这个模块

网关中的拦截器和SpringMVC的拦截器完全不一样，如果这样全局配置也会给网关配置拦截器，会报错

因为网关模块根本就没有MVC拦截器的内核，这样强行给网关配置没有的东西显然不合适

因此，配置的时候就要排除网关模块，这里就要用到@ConditionalOnClass(DispatcherServlet.class)

这个注解条件可以实现筛选，只有拥有DispatcherServlet.class类的模块才能实现配置

但是上面提到，网关模块是没有这个类的，因为这是SpringMVC的核心类

所以，还要在配置类上面加上@ComponentScan({"com.hmall", "com.otherpackage",……})注解
