String ​

1.创建String对象的方式 ​

1.1.直接赋值法 ​

String name = "Tom";

写法:左边变量右边值

1.2.构造方法 ​

构造方法说明
public String()创建空白字符串，不含任何内容
public String(String str)根据传入的字符串创建对象
public String(char[] chs)根据字符串数组创建对象
public String(byte[] chs)根据字节数组创建对象

1.3.构造方法举例 ​

根据字符串创建对象

Java
char[] chs={'a','b','c','d'};
String s4 = new String(chs);
System.out.println(s4);

输出结果是abcd

根据字节数组创建对象

Java
byte[] bytes = {97,98,99,100};
String s5 = new String(bytes);
System.out.println(s5);

输出结果是abcd

因为字节数组会把数字先拿到ASCII表里面去查，查到了就转化成对应的字符输出

2.创建String对象的内存原理 ​

2.1.内存原理图 ​

除了这三个基本的内存区域，还有一个部分:StringTable(串池)

串池专门用来存放字符串，但是只有直接赋值的字符串才会存在串池

比如String str = "abc"这样直接赋值的才会存在串池里面

而String str = new String("abc")这样"new出来的对象"的不会存在串池里面

在JDK7以前，串池在方法区里面，但是JDK7以后挪到了堆内存中

2.2.内存模型分析 ​

以如下代码为例

Java
public class StringDemo{
    public static void main(String[] args){
        String s1 = "abc";
        String s2 = "abc";
    }
}

首先，main方法会加载进栈，然后从上往下执行每一行代码

第一行，用等号直接赋值创建了一个String对象，变量名字叫s1

然后系统会去串池里面找，有没有"abc"这个字符串。但是这里是第一次创建，显然找不到

既然找不到，就会在串池里面创建一个"abc"，并给与一个地址值0x0011，然后把这个地址

值赋值给s1

第二行，还是创建了一个叫s2的变量，然后去串池里面找，发现，已经有"abc"存在了！！！

那么，既然存在了，就直接复用已经存在的abc，而不是新创建一个

这就意味着，s1和s2在内存中的地址值是一样的，都是0x0011！！！

而我们知道，在Java中，"=="比较的是两个对象之间的地址值是否相同，所以，s1 == s2

为true是因为地址值相同，而不是因为内容一样

那么，如果我改动一下上面的代码

Java
public class StringDemo{
    public static void main(String[] args){
        char[] chs = {'a','b','c'};
        String s1 = new String(chs);
        String s2 = new String(chs);
    }
}

同样，main方法进栈，先在堆内存里面开辟一块空间，用来存放chs字符串数组

但是，下面就不一样了，因为s1是"new出来的对象"，所以这次直接在堆内存里面开辟一块空

间，并分配一个地址值"0x0011"，且不会存入串池，下面的s2也是在堆内存里面再开辟一块

空间，分配一个地址值"0x0022"，也不会去查询串池。

这一次，虽然s1和s2内容都是"abc"，但是s1 == s2的值就是false，因为地址值不一样

3.字符串的比较 ​

3.1.==号比较 ​

"=="，如果左右两边都是基本数据类型，比如int，double……那么直接比较两者的值

如果是引用数据类型，比如各种数组，String，Object，那么就是比较地址值

比如这样比较，会输出false

因为，上面的s1是"new出来的对象"，地址值是堆内存中的地址值

下面的s2是直接等号赋值的，是串池中的地址值

两个地方都不一样，地址值能一样吗？而"=="比较的又是地址值，自然就是false

3.2.equals比较 ​

还是这个例子，如果不用==号比较，用equals比较呢？

s1.equals(s2)结果就是true

因为Java源码里面，开发者对String方法中的equals方法进行了重写

本来equals方法是String继承自它的父亲对象"Object"的，Object里面的equals方法还是比较

的地址值。但是重写之后，就变成比较字符串的内容而非地址值了

还有一种比较叫equalsIgnoreCase()，这个方法比较忽略大小写

比如String s1 = "abc" 和 String s2 = "AbC" 这两个用这种方法比较是一样的

3.3.Scanner输出字符串比较 ​

比如，有如下代码

Java
Scanner sc = new Scanner(System.in);
String str1 = sc.next();
String str2 = "abc";
System.out.println(str1 == str2);

最终打印结果是false，两个不相等

因为，在sc.next()这个方法的底层，会把输入的字符串以String的有参构造形式创建

比如输入了一个"abc"，在底层，就相当于String str1 = new String("abc")

上面说过了，一个是"new出来的对象"，一个是直接赋值，地址值显然不一样
