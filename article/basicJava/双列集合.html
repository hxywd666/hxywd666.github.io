双列集合 ​

1.双列集合介绍 ​

比如以商品为例，每个商品都有对应的商品名字(左边这一列)和商品价格(右边这一列)

左边这一列我们称之为键，右边这一列我们称之为值

左边这一列是不能重复的，是唯一的；右边这一列是可以重复的。键和值是一一对应的

这也很好理解，一般不会有两个相同的商品类名，但是商品的价格可能一样

每个键和值既然是一一对应，就是绑定在一起的。这个绑定在一起的整体称之为键值对对象(Entry对象)

所以，双列集合每次要添加一对数据，不能只添加键或者只添加值，单独存在没有意义

上图是Java中所有双列集合之间的关系。Map是双列集合的顶层接口，它的方法可以被所有双列集合继承使用

2.双列集合通用API ​

Java
public class mapDemo {
    public static void main(String[] args) {
        Map<String, String> map = new HashMap<>();

        map.put("a","b");
        map.put("c","d");
        map.put("e","f");

        //返回键a对应的值b
        String result = map.remove("a");
        
        //清空所有的元素
        map.clear();

        //判断键"a"是否存在
        boolean keyResult = map.containsKey("a");

        //判断是否为空
        boolean result = map.isEmpty();

        //集合的长度
        int size = map.size();

        System.out.println(keyResult);
        }
}

对于Map<String, String> map = new HashMap<>()

Map不是类，是接口，不能直接创建接口的对象，只能创建接口实现类的对象！！！

<String, String>，左边是键的类型，右边是值的类型，类型是随意的，不一定是String，这里以String为例

对于put方法:

put方法有个返回值，如果键不存在，那么直接把键值对对象添加到map集合中，方法返回null

在添加数据的时候，如果键是存在的，那么会把原有的键值对对象覆盖，会把被覆盖的值进行返回 比如我再添加一句:map.put("c","g")，那么原来的"c","d"键值对就会被覆盖成新的"c","g"

3.Map集合遍历 ​

3.1.键找值 ​

这种方法就是先把键全部拿出来，再根据每个键寻找对应的每个值

Java
Set<String> keys = map.keySet();
for(String key : keys){
    System.out.println(key);
    String value = map.get(key);
    System.out.println(key + "=" + value);
}

map.keySet()会把所有的键以一个单列集合(类似于数学里面学的那种无序集合)的形式获取出来

然后通过增强for进行遍历，先把每一个键拿到，然后根据键去寻找值，也就是用map.get(key)进行"键找值"

get()方法以键作为参数，返回值是键对应的值，这样就可以遍历所有的键值对了

3.2.键值对 ​

这种方法就是把键值对这个捆绑在一起的整体全部拿出来，然后在单独获取键和值

Java
Set<Map.Entry<String, String>> entries = map.entrySet();
for(Map.Entry<String, String> entry : entries){
    String key = entry.getKey();
    String value = entry.getValue();
    System.out.println(key + "=" + value);
}

Set<Map.Entry<String, String>> entries = map.entrySet()就是一口气拿出全部的键值对

其中.Entry的意思就是键值对，<String,String>左边表示键的类型，右边表示值的类型

entrySet()这个方法就是返回map集合中全部的键值对的

遍历的时候，再用getKey和getValue两个方法分别获取键和值，实现了键值对遍历

3.3.Lambda表达式遍历 ​

Java
map.forEach(new BiConsumer<String, String>() {
    @Override
    public void accept(String key, String value) {
        System.out.println(key + "=" +value);
    }
});

这是原版写法，其中

new BiConsumer<String, String>()是一个匿名内部类，左右两边的String指键和值的类型

这个匿名内部类的作用就是拿到map集合里面的键值对

我们只要重写匿名内部类里面的accept方法，就可以根据我们想要的方式去操作键值对了

还可以根据Lambda表达式的语法进行缩写

Java
map.forEach((key, value) -> {System.out.println(key + "=" +value)});

这种用匿名内部类获取键值对的方式，在底层其实就是用增强for循环来遍历的，本质一样

4.HashMap ​

4.1.HashMap的特点 ​

HashMap的特点是:无序、不重复、无索引

  • 无序:指键值对存入和取出的顺序是不一致的
  • 不重复:键是不能重复的
  • 无索引:键是没有索引的，和数组不同，不能通过索引获取键

4.2.底层原理 ​

HashMap和HashSet的底层是一样的，都是哈希表结构的(数据+链表)

JDK8以后，还在原来的哈希表基础上加上了红黑树

哈希表默认长度是16，存入数据的时候，不是从0索引开始存的，而是根据如下公式:

index = (数组长度 - 1) & 哈希值 这样来计算在数组中的索引

哈希值就是对象的整数表现形式，根据hashCode方法计算出来的

默认情况下，哈希值是根据对象的地址值进行计算的

默认情况下，不同对象的哈希值是不一样的(哈希碰撞除外)

一般情况下，会根据对象的属性值进行计算哈希值，重写hashCode方法

重写后的方法，如果对象的属性值相同，计算出来的哈希值就是一样的

比如计算出来存入的索引是4，那么判断4索引的位置是不是null

如果是null，就直接把元素添加进去

如果不是null，调用equals方法比较属性值，如果属性值一样，就直接舍弃；如果不一样，新元素直接挂在旧元素下面

当数组存了16×0.75=12的时候，数组就会扩容成原先的两倍

如果某个链表长度大于8或者数组的长度大于64，那么链表就会自动转换成红黑树存储

如果集合中存储的是自定义对象，就必须要重写hashCode和equals方法
