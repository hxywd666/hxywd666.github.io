准备工作

1090字约4分钟

2024-09-05

1. 准备一个QQ小号

由于种种因素，现在QQ官方对于QQ机器人的管控是比较严格的，而这里的QQ机器人开发必须依托于一个QQ号来实现。考虑到有被封控的风险，建议不要随便拿自己辛苦练的大号来开发QQ机器人，测试的时候尽量不要选择人数很多的大群进行测试，只要别明目张胆地滥用QQ机器人，一般也不至于被封号

这个小号其实最好不要是新号，QQ等级不宜过低，容易被封。但是实际上问题也不大，测试或者实际使用的时候收敛一些即可。尽可能使用小号，减少被封号的损失

2. 安装Lagrange.OneBot服务器

首先，你需要下载框架官方提供的一个代理服务器，它可以帮你把你本机发出去的请求转发给QQ官方

点击右侧的链接文字即可跳转到下载页面(需要魔法)👉Lagrange.Core

跳转的Github的Release界面之后，选择你操作系统对应的版本，比如我是Windows 64位的操作系统，那么可以选择下图中用红色矩形框起来的那个zip压缩包：

下载之后将这个压缩包解压到你电脑上的某个文件，然后你可以看到一个叫做Lagrange.OneBot.exe的文件，双击这个可执行文件，然后它就会弹出一个命令行窗口并开始执行，有如下截图：

可以发现，程序运行之后，命令行窗口中会展示出一个二维码，这个二维码是用来扫码登录QQ的，因为机器人是需要托管你准备的QQ号的，所以在服务器第一次启动的时候需要你扫码登录QQ号

如果命令行窗口没有二维码，程序执行之后会在同一个文件夹下面生成一个jpg格式的图片，这张图片也是登录用的二维码，你登录的时候扫哪个都行

扫码之后，记得勾选一下免扫码登录的选项，这样只有第一次服务器启动的时候需要扫码，之后都可以直接自动登录

如果你的手机QQ上显示有Linux设备登录了QQ，那么说明登录成功！

3. 更改配置文件

扫码登录完成之后，就可以把命令行窗口先暂时关掉了，然后看一下exe文件所在的文件夹下面，是否和下图一样：

此时我们需要修改 appsettings.json 配置文件，粘贴以下内容进配置文件：

{
    "Logging": {
        "LogLevel": {
            "Default": "Information",
            "Microsoft": "Warning",
            "Microsoft.Hosting.Lifetime": "Information"
        }
    },
    "SignServerUrl": "https://sign.lagrangecore.org/api/sign/25765",
    "MusicSignServerUrl": "",
    "Account": {
        "Uin": 0,
        "Password": "",
        "Protocol": "Linux",
        "AutoReconnect": true,
        "GetOptimumServer": true
    },
    "Message": {
        "IgnoreSelf": true,
        "StringPost": false
    },
    "QrCode": {
        "ConsoleCompatibilityMode": false
    },
    "Implementations": [
        {
            "Type": "ReverseWebSocket",
            "Host": "127.0.0.1",
            "Port": 8080,
            "Suffix": "/onebot/v11/ws",
            "ReconnectInterval": 5000,
            "HeartBeatInterval": 5000,
            "AccessToken": ""
        },
        {
            "Type": "Http",
            "Host": "127.0.0.1",
            "Port": 8081,
            "AccessToken": ""
        }
    ]
}

简单说一下这里的配置，可以看到配置文件中配置了8080和8081两个端口号，其中8080端口号配置的是"反向WebSocket"，而8081端口号配置的是"HTTP"

这样配置有一个好处就是，既可以用后面框架提供的封装好的方法来开发QQBot，也可以通过axios发送Http请求来开发QQBot，更加灵活，因为框架默认是只支持反向WebSocket通信的，不支持HTTP请求！

接下来，保险起见你需要确保本机的8080和8081端口没有被占用，否则会影响服务器的正常启动

提示

检查端口可以摁住Win + R键，在弹出的窗口中输入cmd然后回车

输入命令"netstat -ano | findstr :<端口号>"来检查对应的端口是否被占用

如果你发现对应的端口有进程占用，记住进程的PID，然后输入"taskkill /PID <PID号> /F"来关闭进程

确定端口没有被占用之后双击Lagrange.OneBot.exe，重新启动服务器

如果启动之后看到QQ显示"Linux设备登录"，那么就说明服务器启动成功了，准备工作完成🎉
